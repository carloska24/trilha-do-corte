# Backend API Master

## Trigger

- criar backend
- criar api
- backend profissional
- api rest
- graphql
- servidor
- regras de negocio

## Inputs

### Required

- `requisitos_do_produto`

### Optional

- `tipo_de_api`
- `escala_esperada`
- `integracoes`

## Steps

1. **backend_architect**

   - Task: Definir tipo de API, padrões e camadas.

2. **domain_engineer**

   - Task: Modelar entidades e regras de negócio.

3. **api_engineer**

   - Task: Implementar endpoints, DTOs e validações.

4. **auth_engineer**

   - Task: Implementar autenticação e autorização.

5. **integration_engineer**
   - Task: Preparar integrações externas e contratos.

## Output Deliverables

- Estrutura do backend
- Endpoints documentados
- Autenticação configurada
- Validações e regras de negócio
- Boas práticas de segurança
