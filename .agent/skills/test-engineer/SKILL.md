# Test Engineer

## Trigger

- criar testes
- testar sistema
- testes automatizados
- qualidade
- qa
- testes e2e

## Inputs

### Required

- `componentes_criticos`

### Optional

- `stack`
- `nivel_de_cobertura`
- `pipeline_ci`

## Steps

1. **qa_engineer**

   - Task: Definir estratégia de testes e níveis de cobertura.

2. **unit_test_engineer**

   - Task: Criar testes unitários para regras críticas.

3. **integration_test_engineer**

   - Task: Criar testes de integração entre módulos.

4. **e2e_test_engineer**

   - Task: Criar testes end-to-end para fluxos principais.

5. **quality_guard**
   - Task: Revisar confiabilidade, flakiness e manutenção dos testes.

## Output Deliverables

- Estrutura de testes definida
- Testes unitários
- Testes de integração
- Testes end-to-end
- Boas práticas de qualidade
